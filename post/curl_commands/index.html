バイナリをダウンロードして展開するイディオム 

# e.g. Antropic CLI のインストール
curl -fsSL "https://github.com/anthropics/anthropic-cli/releases/download/v${VERSION}/ant_${VERSION}_linux_amd64.tar.gz" \
    | sudo tar -xz -C /usr/local/bin ant

ファイルを保存せず、ダウンロード結果をパイプで直接 tar に流して /usr/local/bin/ant に配置する。

curl -fsSL の意味

オプション 意味
-f (--fail) HTTP エラー（404, 500 等）で出力を出さず非ゼロ終了。これがないとエラー HTML が tar に流れて壊れる
-s (--silent) プログレスバー・エラーを抑制
-S (--show-error) -s で消えるエラーだけは表示。-sS セットで「進捗は黙れ、失敗は教えろ」になる
-L (--location) リダイレクト追従。GitHub Releases は S3 等へリダイレクトするので必須

tar -xz -C <dir> <file> の意味

要素 意味
-x (--extract) 展開
-z (--gzip) gzip 解凍
-C <dir> 展開先を指定（cd してから展開するのと同等）
<file> tarball から特定ファイルだけ取り出す

注意点

  • set -o pipefail を有効にしないと、curl の失敗がパイプライン全体の終了ステータスに反映されない（bash 推奨）
  • tarball 内のパスが ant_x.y.z_linux_amd64/ant のようにディレクトリ階層を持つ場合は --strip-components=1 が必要
