Why

個人や、グループ開発で様々なシーン 差分確認 の場面がある。
その中で、 いち早く問題特定 ができるようになりたい。
バグ修正時やペアプログラミングの際、はたまたコードレビュー時に鋭くかつ適切に理解ができる効果を期待したいため。

How

[App] SourceGraph

https://github.com/sourcegraph

いち早く問題特定するためには、何といってもコード理解が必須である。
いちいち差分を手作業で見たり、そもそも修正箇所を特定してから差分比較するというときには、 やはりリポジトリ内での検索力が問われる。
そこで、「SourceGraph」なるツールがあることを知った。

[WebUI] URL

URLを加工してBrowser画面上で比較

※ 便宜的にURLノード別に改行している
https://github.com
       /[ユーザー（組織）名]
       /[リポジトリ名]
       /compare
       /[Old]...[New]
  • コミット間
    1. GitHubのリポジトリ頁へ
    2. 「🕐 Commits」ボタンをクリック
    3. コミット一覧より比較したい世代のコミットIDを選択
    4. https://github.com/[ユーザー（組織）名]/[リポジトリ名]/compare/コミットハッシュ値old...コミットハッシュ値new
      • e.g. https://github.com/numpy/numpy/compare/b01a473...4d29079
  • ブランチ間
    1. リポジトリのトップ頁から「Compare」ボタンをクリック
    2. ブランチの比較頁へ遷移
      • e.g. https://github.com/numpy/numpy/compare/master...maintenance/1.1.x
  • 日付間
    1. n日前
      • e.g. https://github.com/numpy/numpy/compare/master@{7.day.ago}...master
    2. 指定日
      • e.g. https://github.com/numpy/numpy/compare/master@{2020-12-01}...master

[WebUI] Link Share

GitHubのWebUI上で、コードやページに対して共有リンクを発行することができる。
修正したい箇所や共有したい箇所に対して、右クリックで発行してシェア。

[CLI] $ git diff 

$ git diff
  • コミット間
    1. 今回コミットと直前 $ git diff HEAD^
    2. 指定コミット同士 $ git diff 変更前のSHA..変更後のSHA
  • ブランチ間
    • 指定ブランチ間 
      $ git diff ブランチA..ブランチB

[CLI] $ git grep
